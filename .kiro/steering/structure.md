# Project Structure

## Directory Organization

```
/
├── app/                    # Next.js App Router pages and components
│   ├── blog/              # Blog posts (nested routes)
│   │   ├── getting-started-guide/
│   │   ├── modbus-protocol-deep-dive/
│   │   ├── modbus-tcp-ip-explained/
│   │   ├── v0-2-0-release/
│   │   └── page.tsx       # Blog index
│   ├── components/        # Shared React components
│   │   └── StructuredData.tsx
│   ├── docs/              # Documentation pages
│   ├── download/          # Download page
│   ├── features/          # Features page
│   ├── globals.css        # Global styles
│   ├── layout.tsx         # Root layout with metadata
│   └── page.tsx           # Homepage
├── public/                # Static assets
│   ├── downloads/         # Downloadable files
│   ├── screenshots/       # Product screenshots (optimized)
│   ├── icon-*.png/webp    # Optimized icons
│   ├── robots.txt
│   └── sitemap.xml
├── scripts/               # Build and optimization scripts
│   ├── optimize-images.js
│   └── postbuild-optimize.js
├── .next/                 # Next.js build output (gitignored)
├── out/                   # Static export output (gitignored)
└── node_modules/          # Dependencies (gitignored)
```

## Routing Convention

- **File-based routing**: Each `page.tsx` in `app/` creates a route
- **Nested routes**: Folders create URL segments (e.g., `app/blog/v0-2-0-release/page.tsx` → `/blog/v0-2-0-release`)
- **Layout inheritance**: `app/layout.tsx` wraps all pages

## Component Organization

- **Shared components**: Place in `app/components/`
- **Page-specific components**: Can be colocated with page files
- **Client components**: Use `"use client"` directive when needed (e.g., for interactivity)

## Asset Management

- **Images**: Store in `public/` directory
- **Optimized images**: Generated by build scripts, stored alongside originals
- **Naming convention**: Responsive images use `-{width}` suffix (e.g., `main-dashboard-640.webp`)

## Styling Patterns

- **Tailwind utility classes**: Primary styling method
- **Responsive design**: Mobile-first with `sm:`, `md:`, `lg:` breakpoints
- **Custom colors**: Use `primary-{50-900}` scale from Tailwind config
- **Semantic HTML**: Use proper heading hierarchy and ARIA labels

## SEO & Metadata

- **Metadata**: Defined in `layout.tsx` and individual page files
- **Structured data**: Use `StructuredData` component for JSON-LD
- **Sitemap**: Static file in `public/sitemap.xml`
- **Robots**: Static file in `public/robots.txt`

## Code Conventions

- **TypeScript**: All components use `.tsx` extension
- **Imports**: Use `@/` alias for workspace root imports
- **Client vs Server**: Default to server components, add `"use client"` only when needed
- **Accessibility**: Include ARIA labels, semantic HTML, and proper alt text
